# Versioning (Normative)

TIR-STD follows Semantic Versioning: MAJOR.MINOR.PATCH

- **MAJOR**: breaking schema/semantics changes
- **MINOR**: backwards-compatible additions
- **PATCH**: clarifications and bug fixes without semantic change

Rule packs (TIR-RP) also follow SemVer and must declare:
- version
- compatibility range (optional)
- last verification date
